Margin Pools · 买方 / 卖方保证金池

买方 / 卖方保证金池 · 只对成交收费的双边手续费模型

只对「成交的名义本金」按 0.1% 收取手续费，未成交不收费，平台与生态 50 : 50 分成。

买方保证金池 + 卖方保证金池，是 NST 期权 / CFD 交易的核心结算层。 所有订单在撮合成交后，才会对名义本金按 0.1% 的双边手续费 收费， 未成交订单不收任何手续费。

  • 成交时，按名义本金的 0.1% 收取手续费（示例：100,000 U 名义本金 → 手续费 100 U）。
  • 买方侧 + 卖方侧手续费集中进入保证金池，用于平台可持续运营和生态激励。
  • 手续费收入分配：50% 作为平台收入， 50% 进入生态奖励池，给节点、veNST、LP 等做长期分红。

工程师：合约层只需要在成交时收取 0.1% 的 fee，并记录 fee 的去向（平台地址 + 生态地址）， 保证未成交没有扣费逻辑；前端这个模块只做规则展示 + 引导按钮。

进入保证金池 · 买方建仓 / 卖方建仓

买方保证金用于支付权利金，卖方保证金用于承担履约责任；成交后统一从保证金池中结算。

买方保证金池 · 买方建仓

买方在下单 / 建仓时，只需要支付权利金 + 必要保证金。 订单只有在成交时才产生 0.1% 的手续费，未成交订单不收任何费用。

  • 适合做「有限亏损、无限获利」的期权 / CFD 策略。
  • 所有买方的手续费，会进入保证金池，用于平台 + 生态的长期分配。

卖方保证金池 · 卖方建仓

卖方在接单 / 建仓时，需要按名义本金缴纳足够保证金，用于未来履约。 订单只有在成交时才扣 0.1% 手续费，未成交挂单不收费。

  • 适合懂对冲 / 有风控能力的专业卖方，赚取权利金收益。
  • 卖方成交产生的手续费，同样按 50% 平台 + 50% 生态的规则分配。

手续费三段式逻辑（工程师必读）：

  1. 只在「成交时」扣除手续费 = 名义本金 × 0.1%，未成交不扣费。
  2. 扣到的手续费自动按 50% : 50% 划分为：平台收入地址 + 生态奖励地址。
  3. 生态奖励地址的资金，再根据规则分给节点 / veNST / LP / 任务积分池，全部上链可查。

Referral Program

推荐计划不是简单拉人头，而是围绕「人数 / 质押金额 / 贡献利润」三维度做空投。 你推荐的用户越「深度使用生态」，你的 NST 空投越可观。

每个新地址通过你的链接首次使用本生态（捐赠 / 节点 / 质押 / 交易）时，自动绑定推荐关系。

三种推荐空投来源：

  • 推荐使用人数：达成一定人数档位，获得一笔空投（例如每 10 人一档）。
  • 推荐质押金额：你推荐的用户在 LP / 对冲池 / 节点的累计质押金额达到档位，送空投。
  • 推荐贡献利润：你推荐的用户带来的实际利润 / 手续费收入达到档位，再送空投。

实际档位（例如 10 人 / 50 人 / 100 人，或 5 万 / 10 万 U 质押）由后台配置，方便后期调节。
